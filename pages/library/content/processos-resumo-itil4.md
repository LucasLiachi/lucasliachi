# Resumo — Framework ITIL 4

## O que é ITIL 4

ITIL 4 (Information Technology Infrastructure Library) é um framework de melhores práticas para **gerenciamento de serviços de TI (ITSM)**. A versão 4, lançada em 2019, integra conceitos de Agile, DevOps e Lean ao modelo tradicional de processos.

**Objetivo central:** criar, entregar e melhorar continuamente serviços de TI que gerem valor para clientes e partes interessadas.

## Sistema de Valor de Serviço (SVS)

O SVS é o núcleo do ITIL 4. Transforma demanda e oportunidade em valor através de:

```
Oportunidade / Demanda
        ↓
   [Cadeia de Valor]  ←→  [Práticas ITIL]
        ↓
     VALOR
       ↑
  (guiado pelos 7 Princípios Orientadores)
  (governado pelo Modelo de Governança)
  (melhorado pela Melhoria Contínua)
```

## Os 7 Princípios Orientadores

| Princípio | Resumo prático |
|---|---|
| **Foque em valor** | Toda ação deve gerar valor para o cliente ou partes interessadas |
| **Comece onde você está** | Avalie o estado atual antes de criar algo novo |
| **Progrida iterativamente com feedback** | Entregue valor em incrementos, ajuste com aprendizado |
| **Colabore e promova visibilidade** | Trabalhe junto, compartilhe informação, evite silos |
| **Pense e trabalhe holisticamente** | Considere o sistema todo, não apenas partes isoladas |
| **Mantenha simples e prático** | Elimine etapas que não agregam valor |
| **Otimize e automatize** | Use recursos humanos para o que não pode ser automatizado |

## Cadeia de Valor de Serviço (CVS) — 6 atividades

| Atividade | Papel |
|---|---|
| **Planejar** | Garantir entendimento e direção compartilhados da visão, melhorias e planos |
| **Melhorar** | Melhoria contínua de práticas, serviços e produtos |
| **Engajar** | Boa compreensão das necessidades das partes interessadas |
| **Projetar e Transicionar** | Serviços e práticas que atendem às expectativas de qualidade |
| **Obter/Construir** | Disponibilização de componentes de serviço |
| **Entregar e Suportar** | Entrega e suporte dos serviços conforme especificado |

## Conceitos-chave

| Termo | Definição |
|---|---|
| **Serviço** | Meio de co-criar valor, facilitando resultados sem gestão dos custos e riscos pelo cliente |
| **Valor** | Percepção de benefícios, utilidade e importância |
| **Utilidade** | Funcionalidade oferecida ("o que faz") — fit for purpose |
| **Garantia** | Confirmação de que o serviço atende requisitos — fit for use |
| **Resultado** | Resultado para o cliente/stakeholder habilitado pelo serviço |
| **SLA** | Acordo de Nível de Serviço — entre provedor e cliente |
| **OLA** | Acordo de Nível Operacional — interno, entre equipes |
| **UC** | Contrato de apoio — com fornecedor externo |
| **RACI** | Responsible, Accountable, Consulted, Informed — modelo de papéis |

## Diferenças ITIL v3 vs ITIL 4

| Aspecto | ITIL v3 | ITIL 4 |
|---|---|---|
| Estrutura | 5 livros (ciclo de vida) | SVS + Cadeia de Valor |
| Terminologia | Processos | Práticas |
| Integração | Focado em ITSM | Inclui Agile, DevOps, Lean |
| Quantidade | 26 processos | 34 práticas |
| Foco | Operações e eficiência | Valor e co-criação |
