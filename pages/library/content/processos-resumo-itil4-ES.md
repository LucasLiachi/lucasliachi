# Resumen — Framework ITIL 4

## Qué es ITIL 4

ITIL 4 (Information Technology Infrastructure Library) es un framework de mejores prácticas para la **gestión de servicios de TI (ITSM)**. La versión 4, lanzada en 2019, integra conceptos de Agile, DevOps y Lean al modelo tradicional de procesos.

**Objetivo central:** crear, entregar y mejorar continuamente servicios de TI que generen valor para clientes y partes interesadas.

## Sistema de Valor de Servicio (SVS)

El SVS es el núcleo de ITIL 4. Transforma demanda y oportunidad en valor a través de:

```
Oportunidad / Demanda
        ↓
   [Cadena de Valor]  ←→  [Prácticas ITIL]
        ↓
     VALOR
       ↑
  (guiado por los 7 Principios Guía)
  (gobernado por el Modelo de Gobernanza)
  (mejorado por la Mejora Continua)
```

## Los 7 Principios Guía

| Principio | Resumen práctico |
|---|---|
| **Enfócate en el valor** | Toda acción debe generar valor para el cliente o partes interesadas |
| **Empieza donde estás** | Evalúa el estado actual antes de crear algo nuevo |
| **Progresa iterativamente con retroalimentación** | Entrega valor en incrementos, ajusta con aprendizaje |
| **Colabora y promueve la visibilidad** | Trabaja en conjunto, comparte información, evita silos |
| **Piensa y trabaja holísticamente** | Considera todo el sistema, no solo partes aisladas |
| **Mantén la simplicidad y practicidad** | Elimina pasos que no agregan valor |
| **Optimiza y automatiza** | Usa recursos humanos para lo que no puede automatizarse |

## Cadena de Valor de Servicio (CVS) — 6 actividades

| Actividad | Rol |
|---|---|
| **Planificar** | Asegurar comprensión y dirección compartidas de la visión, mejoras y planes |
| **Mejorar** | Mejora continua de prácticas, servicios y productos |
| **Involucrar** | Buena comprensión de las necesidades de las partes interesadas |
| **Diseñar y Transicionar** | Servicios y prácticas que cumplen las expectativas de calidad |
| **Obtener/Construir** | Disponibilización de componentes de servicio |
| **Entregar y Soportar** | Entrega y soporte de los servicios según lo especificado |

## Conceptos clave

| Término | Definición |
|---|---|
| **Servicio** | Medio de co-crear valor, facilitando resultados sin que el cliente gestione costos y riesgos |
| **Valor** | Percepción de beneficios, utilidad e importancia |
| **Utilidad** | Funcionalidad ofrecida ("qué hace") — fit for purpose |
| **Garantía** | Confirmación de que el servicio cumple los requisitos — fit for use |
| **Resultado** | Resultado para el cliente/stakeholder habilitado por el servicio |
| **SLA** | Acuerdo de Nivel de Servicio — entre proveedor y cliente |
| **OLA** | Acuerdo de Nivel Operacional — interno, entre equipos |
| **UC** | Contrato de apoyo — con proveedor externo |
| **RACI** | Responsible, Accountable, Consulted, Informed — modelo de roles |

## Diferencias ITIL v3 vs ITIL 4

| Aspecto | ITIL v3 | ITIL 4 |
|---|---|---|
| Estructura | 5 libros (ciclo de vida) | SVS + Cadena de Valor |
| Terminología | Procesos | Prácticas |
| Integración | Enfocado en ITSM | Incluye Agile, DevOps, Lean |
| Cantidad | 26 procesos | 34 prácticas |
| Foco | Operaciones y eficiencia | Valor y co-creación |
