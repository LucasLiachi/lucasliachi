# Template para Architecture Decision Record (ADR)

Este arquivo serve como um modelo para registrar as decisões arquiteturais do projeto Antigravity.

Copie este template para um novo arquivo na pasta `docs/architecture/decisions/` com o nome `ADR-XXX-nome-da-decisao.md` sempre que uma nova decisão importante for tomada.

---

# Título: ADR-XXX: [Nome ou Breve Descrição da Decisão]

**Status:** [Proposto | Aceito | Rejeitado | Obsoleto]
**Data:** [YYYY-MM-DD]

## Contexto
Qual o problema, cenário ou necessidade que motivou esta decisão? Explique o que precisa ser resolvido e quais os desafios (ex: *Precisamos processar tarefas pesadas de forma assíncrona* ou *O carregamento inicial da página está lento*).

## Decisão
Qual é a decisão técnica adotada? Seja claro e direto (ex: *Utilizaremos [Tecnologia X / Padrão Y] para resolver a necessidade descrita*).

## Consequências
O que acontece a partir de agora? Quais são as implicações boas e ruins?
- *Ex: Aumenta a complexidade de infraestrutura.*
- *Ex: Garante resiliência e escalabilidade.*
- *Ex: Será necessário refatorar a classe X.*
